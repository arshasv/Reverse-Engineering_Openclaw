## **3\. Implementation Plan: Login, Rate-Limiting & Extraction**

To ensure you don't trigger "Bot Detection" or affect the actual site's performance, use the following configuration in your OpenClaw/Thor script:

#### **A. Intelligent Rate-Limiting**

Configure your agent to mimic a high-speed human, not a machine:

* **Per-Minute Call Limit:** Set a maxConcurrent: 1 and a delayBetweenActions: 2000ms (2 seconds).  
* **Human Jitter:** Add a random offset (±500ms) to every click so the timing isn't identical every time.  
* **Rate Limit Header:** If using OpenClaw's internal API, set maxRequests: 30/min in the openclaw.json security block.

#### **B. The "Full App" Capture Workflow**

Once logged in, instruct the agent to:

1. **Map the DOM:** Use openclaw browser snapshot to assign numeric references to all buttons.  
2. **Sequential Screenshots:**  
   * Maps to a module (e.g., /scheduling).  
   * Take Screenshot (Full Page).  
   * Click every primary button once to trigger modals (e.g., "Add New Job").  
   * Take Screenshot of the modal to capture all input fields and validation logic.
